A **pass** is the card, ticket, or credential a customer holds in their Apple Wallet or Google Wallet. The **Passes** page lists every pass issued for your company's programs, so you can see who's using them and look up any individual pass.

![Passes list](/images/portal/passes/list.png)

## The passes list

Open **Passes** in the sidebar. Each row shows:

| Column | Meaning |
|---|---|
| **ID** | The pass identifier (shortened — hover to see the full ID) |
| **Status** | The pass's current lifecycle status, like active or voided |
| **Wallet** | Whether it's an Apple Wallet or Google Wallet pass |
| **Cardholder** | The customer's name, when provided |
| **Program** | The program the pass belongs to — click it to open the program |
| **Created** | When the pass was issued |

Click any row to open the [pass detail page](/passes/lifecycle), where you can follow the pass's status history, wallet installations, and transactions.

If your company hasn't issued any passes yet, the list is empty — start with [issuing a pass](/passes/issuing).

## What you can do from here

- **Issue pass** — create a new pass for a customer. See [issue a pass](/passes/issuing).
- **Open a pass** — click a row to inspect a single pass, send it a notification, or [void it](/passes/lifecycle#void-a-pass).
